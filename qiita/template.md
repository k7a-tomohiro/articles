# はじめに

この記事を書こうと思ったきっかけ・課題感を1〜3文で。読者が「自分のことだ」と共感できるよう書く。
完成物のGIF/スクショをここに貼ると◎ — ゴールが一目でわかりいいねが増えやすい
AIの利用有無についても記載。

**読者の共感を得る。**

「〇〇で詰まった」「〇〇を調べたら日本語記事がなかった」など、書こうと思ったきっかけを素直に書く。読者が「自分も同じ状況だ」と思えると、最後まで読み続けてもらいやすい。

- Good: Dockerfileのベストプラクティスを調べると英語記事ばかりだったので日本語でまとめた
- Good: 社内でよく聞かれる質問なので一度まとめておきたかった
- 避けたい: この記事ではXXについて解説します（理由なし）

**完成物・ゴールを先に見せる**

「この記事を読むと何ができるようになるか」を冒頭で宣言する。GIF・スクリーンショット・デモURLがあれば冒頭に貼ると◎。視覚でゴールが伝わると離脱率が下がる。

**対象読者の明示**

「Pythonの基礎は知っている人向け」「初めてDockerを触る人向け」など、誰のための記事かを1行で書く。対象を絞るほど刺さる読者が増える。

## 対象読者・前提環境

「〇〇を知っている人向け」と明示。バージョン・OS・ツールをコードブロックで列挙。

```
- Python 3.12 / Node 20 / macOS 14 のように箇条書きが読みやすい
```

**環境情報は具体的に**

バージョンの曖昧な記事はすぐ陳腐化する。OS・ランタイム・主要ライブラリのバージョンをコードブロックに列挙する。

- Good: - macOS 14.4 / Ubuntu 22.04 - Python 3.12.3 - FastAPI 0.111.0

**前提知識の線引き**

「〇〇は説明しない」と明示すると、読者が記事の範囲を即座に判断できる。「Gitの基本操作は知っている前提で進めます」のように書くだけで十分。

**参照記事へのリンク**

前提知識に不安な読者のために「Gitを知らない方はこちら」と入門記事へのリンクを添えると親切。記事の信頼性も上がる。

## 概要・全体像

**地図を先に渡す**

長い記事ほど「自分が今どこにいるか」がわからなくなる。アーキテクチャ図・フロー図・箇条書きの概要を最初に提示して、読者に地図を渡してから詳細に入る。

**なぜその技術を選ぶか**

「なぜXXではなくYYを使うか」「既存の方法と何が違うか」をひと言触れると、読者の「なぜ読むべきか」という疑問に答えられる。モチベーションが高まり最後まで読んでもらいやすい。

**ディレクトリ構成の提示**

コードを含む記事では、最終的なディレクトリ構成をここに貼ると読者が全体を把握しやすい。

# 手順（Step 1〜N）

1ステップ1見出し。コードブロックには言語指定＋ファイル名を付ける。ハマりやすいポイントは :::note warn で補足。
コードは差分（diff）より完全なスニペットを貼る方がコピーされやすい。

**1ステップ＝1見出し**

1つの見出しに複数の作業を詰め込まない。「インストール → 設定 → 動作確認」はそれぞれ別の見出しに分ける。右側の目次が読者のナビゲーターになる。

**コードブロックの書き方**

言語指定＋ファイル名を必ずつける。完全に動くスニペットを貼る（部分省略は読者を混乱させる）。変更箇所にはコメントで `# ここを変更` と書くと親切。

**ハマりポイントを先回りして書く**

自分が詰まった箇所、読者が間違えやすい箇所を `:::note warn` や `:::note alert` で強調する。「ここで〇〇エラーが出た場合は△△する」と書くだけで記事の価値が大きく上がる。

**「なぜそうするか」を添える**

コマンドや設定値を羅列するだけでなく、「このオプションをつける理由は〇〇です」と理由を書く。理解して使えるようになる記事は長期的にいいねされ続ける。

# 動作確認・デモ

「ここまでやれば動く」という確認コマンドや出力例。スクリーンショット1枚あると安心感が上がる。

**再現可能な確認手順**

「curl localhost:8000 で以下のレスポンスが返ってくれば成功」のように、コマンドと期待される出力をセットで書く。コピペで確認できると読者の安心感が大きく違う。

**スクリーンショットの活用**

ターミナルの出力やUIのスクショを1枚貼るだけで「本当に動いた」という信頼感が生まれる。

# まとめ

3〜5行で要点を箇条書き。
「今回やったこと」を3〜5行の箇条書きでまとめる。読み飛ばしで来た読者がここだけ読んでいいねを押すことも多い。
「さらに詳しくは公式ドキュメントへ」「応用編は別記事で書く予定」と書くと記事に続きを感じさせられる。シリーズ化するとフォロワーが増えやすい。
